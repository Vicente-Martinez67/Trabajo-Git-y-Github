1.	¿Qué es Git?
- Git es un sistema de control de versiones  que permite registrar los cambios hechos en el codigo.

2.	¿Qué es GitHub?
- GitHub es una aplicacion en linea que permite almacenar repositorios en la nube. Ademas facilita el trabajo colaborativo por ue se puede usar herramientas como: ramas,pull request y control de versiones.

3.	¿Cuál es la diferencia entre Git y GitHub?
- La diferencia es que Git administra el historial de cambios de un proyecto en forma local, mientras que GitHub es un servicio que aloja repositorios Git en internet para colaborar con otras personas.

4.	¿Qué es un repositorio?
- Un repositorio es el lugar donde se alamcenan todos los archivos de un proyecto junto a su historial de cambios, permitiendo mantener organizado las distintaas versiones del desarrollo del proyecto.

5.	¿Qué función cumple git add?
- La funcion que cumple git add es de agregar los archivos modificados al staging o area de preparacion indicando que esos cambios seran incluidos en el proximo commit.


6.	¿Qué función cumple git commit?
- La funcion que cumple git commit es que guarda un registro de los cambios preparados, acompañado de un mensaje para describir las modificaciones realizadas.

7.	¿Qué función cumple git push?
- La funcion que cumple git push es de enviar los commits realizados en el repositorio local hacia el repositorio remoto, que es GitHub.

8.	¿Qué función cumple git pull?
- La funcion que cumple git pull es de descargar los cambios mas recientes desde el repositorio remoto y los integra con la copia local del proyecto.

9.	¿Cuáles son las ventajas del control de versiones?
- Las ventajas del control de versiones es de:
    -permitir recuperar versiones anteriores del proyecto.
    -Facilitar el trabajo en equipo.
    -Mantiene un historial de todos los cambios realizados.
    -Reduce el riesgo de perdida de cualquier informacion.
